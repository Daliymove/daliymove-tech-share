## 低摩擦比功能丰富更重要

个人知识库最怕的不是功能不够，而是每次记录都要先和工具搏斗。Markdown 的好处在于它足够朴素：打开文件，写下标题、正文、列表、代码块，就能完成一次记录。

当工具足够轻，写作就更容易变成习惯。长期来看，稳定输出比偶尔整理出一个漂亮系统更有价值。

## 文件就是边界

Markdown 文件天然拥有清楚的边界。每篇文章都是一个独立文件，迁移、备份、版本管理都很直接。

```ts
const note = {
  title: "清晰的文件边界",
  format: "markdown",
  portable: true,
};
```

这样的结构也很适合 Git：你能看到每次修改的差异，回溯一段想法是怎样变化的。

## 元数据让内容可组织

一篇文章除了正文，还需要一些描述自己的信息，比如标题、发布时间、分类和标签。Frontmatter 就是一个轻量的元数据入口。

| 字段 | 用途 |
| --- | --- |
| title | 页面标题与搜索标题 |
| description | 摘要与 SEO 描述 |
| category | 建立主要知识分类 |
| tags | 串联跨主题线索 |

## 给未来的自己留路

Markdown 的最大优势是可迁移。今天可以用 Astro 渲染成本地博客，明天可以迁移到别的静态站点框架，甚至直接作为纯文本归档。内容不被平台锁住，心里会安稳很多。

> 好的知识系统不是一开始就完美，而是能陪你慢慢长大。
