# DesignBrief — daliyMove 技术分享站首页重构

> 基线文档。后续所有设计系统、线框、原型、QA 均引用本文档。

## 1. 产品定位
个人技术分享站，作者 **daliyMove**。内容以前端、工程实践、工具方法、学习方法、个人系统为主，记录"每天前进一点"的思考与复盘。本质是**长期主义的内容沉淀容器**，不是资讯流，也不是产品落地页。

## 2. 目标用户
- 同行开发者 / 技术学习者，会顺着"前端、Astro、学习方法"等关键词进来。
- 也包括未来的作者本人（翻自己旧文）。
- 阅读场景：桌面端深度读 + 移动端碎片化读。

## 3. 页面唯一目标
**让访客在 5 秒内理解"这是一个持续更新的、有观点的技术笔记站"，并顺势开始读第一篇文章。**
次级目标：展示内容组织的清晰度（分类 / 标签），降低"要不要关注"的决策成本。

## 4. 现有问题（重构动因）
1. 首屏渐变使用 `leaf → blue → violet`，命中禁止清单（紫色 / 蓝紫渐变）。
2. 正文字体为 Inter，命中禁止清单（默认字体）。
3. 右侧"Tech Map"气泡图为视觉噪音，与标题抢视线，且不传递真实信息。
4. "置顶文章"与"最近更新"内容重叠，拉长页面、稀释层级。
5. 侧边栏（分类 / 标签）卡片化过重，读起来像后台模块。

## 5. 设计方向（已选定）
**Direction A —「林间笔记 / 有机手账风」**
保留站点已有的"森林 / 生长"气质，去掉紫蓝渐变与默认字体，把底色暖化为纸感米白，主色收敛为苔藓绿，辅以克制的暖木色。标题用有性格的衬线，正文用非默认的人文无衬线。签名元素用一条"林间小径"式的脉络线替代气泡图。

## 6. 输出格式与精度
- 交付物：完整设计系统（DesignSystemManifest）+ 3 版低保真线框 + 自包含高保真 HTML 原型（含深浅色 / 响应式 / 完整交互状态）+ QAReport + 交付说明。
- 原型精度：可点击、可交付前端实现的工程级 HTML/CSS，使用真实文章内容（非占位文本）。
- 技术栈：纯 HTML + CSS（原型独立，不耦合现有 Astro/Tailwind，便于评审与移植）。
